# Shared Review Logs

> Cross-cutting reviews that aren't tied to a specific task stage.
> For stage-specific reviews, use `{stage}/review.md` inside the task folder.

## When to Use This Folder

- Full repo reviews (architecture audit, code quality sweep)
- Cross-task analysis (dependency audit, security review)
- External agent reports

## Naming

```
{topic}.md
```

Examples:
- `full_repo_review.md`
- `security_audit.md`
- `dependency_review.md`
