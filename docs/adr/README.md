# Architecture Decision Records (ADR)

> **Why decisions were made.** Each ADR captures the context, options considered,
> and rationale for a significant technical decision.

## Format

Use `000-template.md` as the starting point for each ADR.

## Naming

`NNN-short-title.md` — e.g., `001-chose-redis-streams.md`, `002-rejected-langraph.md`

## When to Write an ADR

- Choosing a framework, library, or tool
- Changing architecture (adding a layer, splitting a service)
- Rejecting an approach (document WHY so future agents don't re-propose it)
- Any decision that affects multiple files or future work
